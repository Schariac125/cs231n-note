## 1.线性代数的本质

**到底什么是线性无关和相关**：如果现在有两个向量，这两个向量共线，那么他们数乘特定的值之后相加得到的依然是他们那么维度的向量，但如果不共线呢？那么他们相加就可以得到更高维度的向量，例如线变成面这一类的。也就是说，如果说几个向量都为他们数乘特定值之后相加升维做出了贡献，那么就说他们是线性无关的，那其实这样子想的话，最大线性无关组的意义也就很明了了

**什么又是线性变化**：线性变化有两个非常重要的点，拿二维的平面直角坐标系来举例，经过线性变换之后原点不可以发生改变，原本在这个坐标系上两点连成的任何一条线都不能变成弯曲的，可以理解为这就是一种保持平行且等距的变换

**矩阵的本质是什么**：事实上，我们可以把矩阵看作是由几个向量组成的组合。而矩阵则代表着如何操纵空间，可以理解为它是对基向量的变换。

**矩阵的乘积可以怎么理解**：如果把矩阵的乘法看作是一个线性变换的操作，假设乘上一个矩阵就是一次线性变换，那么如果我们要进行多次线性变换，就需要乘上多个特定变换的矩阵。但事实上，这些矩阵理论上讲都可以复合成一个矩阵，即使用一个矩阵就可以完成所有操作，这个复合矩阵就是前面所有矩阵的总影响。两个矩阵相乘就是两种线性变换相继作用，但实际上可以合为一种

## 2.图像分类

图像的原始表达形式是一个表示亮度值的三维数组，而图像分类器就是基于这个三维数组来操作的

> 一个好的图像分类模型必须对这些变化的所有交叉乘积保持不变，同时保持对类间差异的敏感性。

事实上，我们直接去编写一个识别特定物体的算法并不现实，更现实的方法是把计算机当作一个学生然后通过告诉它这张图是什么，来教会它学习。而这种算法会查看这些实例并且分析他们的特征最后学会。这种依赖于带标签图像的训练数据集的方法被称为数据驱动方法。毕竟这世上物体太多，要每一个都去针对性的编写算法去识别这很抽象而且局限性很强。

这个过程是这样的

- 输入：输入包含了一组N张图像，每一张图像都被标记为K个不同类别之一，这就是训练集
- 学习：模型分析并理解图像特征
- 评估：最后拿一组不存在于数据集中的新图像来评估

### 最近邻分析器（NN）

最近邻分析器其实是一个不是很常用的东西，但是可以帮助我们理解图像分类问题的基本方法就是了

如果要比对几张图像最直观的想法其实是去一个一个比对像素，然后求出综合，一个比较合理的选择是采用曼哈顿距离
$$
d_1 (I_1, I_2) = \sum_{p} \left| I^p_1 - I^p_2 \right|
$$
下面是cs231n的笔记示例代码

```python
import numpy as np
from data_utils import load_CIFAR10
from tqdm import tqdm

Xtr, Ytr, Xte, Yte = load_CIFAR10("data/cifar-10-batches-py")
Xtr_rows = Xtr.reshape(Xtr.shape[0], 32 * 32 * 3)
Xte_rows = Xte.reshape(Xte.shape[0], 32 * 32 * 3)


class nn:
    def __init__(self):
        pass

    def train(self, X, y):
        self.Xtr = X
        self.ytr = y

    def predict(self, X):
        num_test = X.shape[0]
        Ypred = np.zeros(num_test, dtype=self.ytr.dtype)

        for i in tqdm(range(num_test), desc="正在分类图片"):
            distance = np.sum(np.abs(self.Xtr - X[i,:]), axis = 1)
            min_index = np.argmin(distance)
            Ypred[i] = self.ytr[min_index]

            
        return Ypred


if __name__ == "__main__":
    n = nn()
    n.train(Xtr_rows, Ytr)
    print("开始预测...")
    Yte_predict = n.predict(Xte_rows[:100, :])
    accuracy = np.mean(Yte_predict == Yte[:100])
    print("accuracy: %f" % (np.mean(Yte_predict == Yte[:100])))
```

### 线性分类器

学完KNN其实就能发现它真的很不好用，所以我们需要引入一种更加强大的图像分类方法，这个方法包含两个部分

- 一个将原始数据映射到类别分数的评分函数
- 以及一个量化预测分数与真实标签之间一致性的损失函数

#### 从图像到标签分数的参数化映射

这个方法的首要工作就是去定义一个评分函数，把这个图像的像素值映射到每个类别的置信度分数。

> 如前所述，假设有一个训练图像数据集 xi∈RD ，每个图像都对应一个标签 yi 。其中 i=1…N 且 yi∈1…K 。也就是说，我们有 N 个样本（每个样本的维度为 D），以及 K 个不同的类别。例如，在 CIFAR-10 数据集中，我们有一个包含 N = 50,000 张图像的训练集，每张图像有 D = 32 x 32 x 3 = 3072 个像素，K = 10，因为共有 10 个不同的类别（狗、猫、汽车等）。现在，我们将定义评分函数 f:RD↦RK ，将原始图像像素映射到类别分数。

从最简单的函数入手，线性映射
$$
f(x_i, W, b) =  W x_i + b
$$
在上述方程中，我们假设图像 xi 的所有像素都被展平为一个形状为[D x 1]的列向量。矩阵 W（大小为[K x D]）和向量 b（大小为[K x 1]）是该函数的参数。在 CIFAR-10 数据集中， xi 包含第 i 张图像的所有像素展平后的单个[3072 x 1]列向量，W 是[10 x 3072]矩阵，b 是[10 x 1]向量，因此有 3072 个数值（原始像素值）输入函数，并输出 10 个数值（类别得分）。W 中的参数通常被称为权重，而 b 被称为偏置向量，因为它影响输出得分，但不与实际数据 xi 交互。然而，你经常会听到人们交替使用权重和参数这两个术语。

但是要注意的是

- 在上述的情况里，单个矩阵乘法事实上是去评估十个独立的分类器，最后被保存为一个[10 * 1]的矩阵，每个数值就是对应的类别
- 输入数据是给定的，但是参数W和参数b是可以由我们自己规定的，我们的目标就是通过设置这些参数找出最好的结果
- 这个方法的优点在于，训练数据只用来确定和学习参数W和参数b，一旦训练完成之后我们就可以直接把这些训练集全部丢掉，这可比KNN好使多了
- 卷积神经网络也是通过上述的方法把图像像素映射成分数，但是映射函数f更加复杂，原理差不多

线性分类器通过对齐三个颜色通道的所有像素值进行加权求和来得到某个图像的分数，根据我们设置的参数我们可以使得分类器对某些特定的颜色产生偏好或者厌恶（这就是核心了），如果说一个图像的边缘有大量的蓝色，那么船舶的可能性就会上升，如果有大量的绿色什么的，类似于草地亦或者是森林的可能性就会上升。而其他的颜色通道上面就会有负权重

> 将图像类比为高维空间中的点。由于图像被拉伸为高维列向量，我们可以将每张图像解释为该空间中的一个点（例如 CIFAR-10 数据集中的每张图像都是 32x32x3 像素构成的 3072 维空间中的一个点）。同理，整个数据集就是一组带标签的点集。

#### 损失函数

损失函数是用来衡量我们对结果的不满意程度的。直观一点说，我们在分类训练数据方面表现不好，损失就会很高，反之就会很低

定义损失函数有非常多种方式，我们先介绍第一种常用的损失函数

**多类支持向量机损失（SVM）**

> 回想一下，对于第 i 个示例，我们得到了图像 xi 的像素和指定正确类别索引的标签 yi 。得分函数接收像素并计算类别得分的向量 f(xi,W) ，我们将简写为 s （得分的缩写）。例如，第 j 个类别的得分是第 j 个元素： sj=f(xi,W)j 。那么，第 i 个示例的多类 SVM 损失形式化定义如下：

$$
L_i = \sum_{j\neq y_i} \max(0, s_j - s_{y_i} + \Delta)
$$

> 示例。让我们通过一个例子来拆解这个过程，看看它是如何运作的。假设我们有三个类别，它们获得的分数分别为 s=[13,−7,11] ，并且第一个类别是真实类别（即 yi=0 ）。同时假设 Δ （一个我们稍后会详细讨论的超参数）为 10。上述表达式对所有不正确的类别（ j≠yi ）进行求和，因此我们得到两项：
> $$
> L_i = \max(0, -7 - 13 + 10) + \max(0, 11 - 13 + 10)
> $$
> 可以看到第一项为零，因为[-7 - 13 + 10]得到负数，随后通过 max(0,−) 函数阈值化为零。这对样本的损失为零，因为正确类别的分数（13）比错误类别的分数（-7）至少高出边界值 10。实际上差值达到 20，远大于 10，但 SVM 只关心差值是否至少达到 10；超出边界的额外差值通过 max 运算被限制为零。第二项计算[11 - 13 + 10]得到 8。这意味着即使正确类别的分数高于错误类别（13 > 11），但未能达到期望的边界值 10。实际差值仅为 2，因此损失值为 8（即需要增加多少差值才能满足边界条件）。总结来说，SVM 损失函数要求正确类别 yi 的分数至少比错误类别分数高出 Δ （边界值）。若未满足此条件，就会累积损失。

在使用线性评分函数的情况下可以把损失函数改写为以下的等价形式
$$
L_i = \sum_{j\neq y_i} \max(0, w_j^T x_i - w_{y_i}^T x_i + \Delta)
$$


这个时候问题又出现了，如果我们有一个数据集和一组参数W，能够正确分类每个样本。这组参数W并不是唯一的。如果某些参数W可以正确分类所有样本，那么这些参数的任意倍数同样可以损失为0

为了消除这种歧义，我们可以在损失函数中添加一个正则化惩罚项R(W)来实现抑制大权重
$$
R(W) = \sum_k\sum_l W_{k,l}^2
$$
有了正则化之后，完整的多累SVM损失就变成了这样
$$
L =  \underbrace{ \frac{1}{N} \sum_i L_i }_\text{data loss} + \underbrace{ \lambda R(W) }_\text{regularization loss} \\\\
$$
好了，看到这里绝对还是一头雾水，那么简单解释一下这到底是一个什么样的过程。

损失函数到底是干什么用的，可以把它理解为一个检测器，我们输入一个图像进去，明明是猫，模型却打了很高的分数给狗，那么这个时候的差值就是损失函数的值，这个值越大，代表着模型错的越离谱。同样的，我们在那个函数里面还加入了另外的一个Δ，这个其实就是我们的一个容忍范围，如果Δ比实际的差值大，那么我们就可以容忍它，比较佛系摆烂就是了

```python
#X是数据集，y是正确标签
def svm_loss_vectorized(W, X, y, reg):
    """
    全向量化实现 SVM Loss
    """
    num_train = X.shape[0]
    
    # 第一步：算出所有类别的得分 (N, C)
    scores = X.dot(W)
    
    # 第二步：提取每个样本对应的“正确类得分”
    # 我们需要 scores[0, y[0]], scores[1, y[1]]...
    # 使用 np.arange 和 y 作为索引，得到一个 (N,) 的向量，然后用 [:, None] 变成 (N, 1) 方便广播
    correct_class_scores = scores[np.arange(num_train), y][:, None]
    
    # 第三步：计算 Margin (N, C)
    # 利用广播：(N, C) - (N, 1) + 1
    margins = np.maximum(0, scores - correct_class_scores + 1)
    
    # 第四步：修正——正确类别那一部分的 Margin 应该强制设为 0
    # 因为公式里规定 j != y[i]，但广播的时候把正确类也算进去了（结果固定是 1）
    margins[np.arange(num_train), y] = 0
    
    # 第五步：求和并取平均
    data_loss = np.sum(margins) / num_train
    
    # 第六步：别忘了那个“防止过拟合”的正则化项！
    reg_loss = reg * np.sum(W * W)
    
    loss = data_loss + reg_loss
    
    return loss
```



#### Softmax分类器

事实上这种分类器的底层实现和线性分类器完全没有任何差别，区别只在于他们解释分数和损失函数的定义不同

Softmax分类器认为原始的分数太过"狂野"，他必须修正，必须把它们归一化到0和1之间，而且总和需要是1

映射函数依然保持着不变，但是我们需要用这样的形式来重新定义损失函数
$$
L_i = -\log\left(\frac{e^{f_{y_i}}}{ \sum_j e^{f_j} }\right) \hspace{0.5in} \text{or equivalently} \hspace{0.5in} L_i = -f_{y_i} + \log\sum_j e^{f_j}
$$
在上面的那个代码里面只需要这样子改一下

```python
# --- 线性 SVM 逻辑 ---
margins = np.maximum(0, scores - correct_class_scores + 1)
loss = np.sum(margins)

# --- Softmax 逻辑 ---
# 1. 为了数值稳定性，先减去每行的最大值（防止 exp 爆炸）
scores -= np.max(scores, axis=1, keepdims=True)
# 2. 计算指数
exp_scores = np.exp(scores)
# 3. 计算概率
probs = exp_scores / np.sum(exp_scores, axis=1, keepdims=True)
# 4. 计算损失：只关注正确类别的概率
correct_logprobs = -np.log(probs[np.arange(num_train), y])
loss = np.sum(correct_logprobs) / num_train
```

计算概率是后续深度学习的基石，这让我们可以更好的解释每个类别的置信度



**总结**：事实上线性分类器就是使用我们给它的训练集去训练出来一个模板（这个模板就是公式里的W矩阵的其中一列）。而预测过程就是我们输入一个图像，利用线性分类器的那个公式去算出来一个合理的分数，然后看和哪个模板之间的分数最高，也就是相似度最高，那么我们就认为应该是这样的。损失函数又是什么意思，就像我们给它一张猫的图像，他却给狗的那边打了高分，这就是产生了偏差。这个时候与猫之间的差值就是损失函数，而我们通常会人为去设定一个阈值，防止过拟合，也就是说我们有一个容忍度。而设立一个阈值很多时候其实是不够的，因为模型有着不断去提升自己以达到分数百分百契合的优势。那这个时候就有人要问了：主播模型损失小不是好事吗？很常见的错觉。模型损失太小也有可能是意味着过拟合了，也就是模型太过于关注细节。我前面说的会哈气的圆头耄耋和不会哈气的爵士好耄就是这样的。这时候就要使用正则化去抑制和惩罚这种过拟合的趋势，保证模型具有通用性。而softmax分类器在底层原理上和普通的线性分类器不存在明显的差别，只是它们在计算结果有所不同。softmax分类器采用了归一化，将每一个分数都化成一个小于等于1的数值，代表概率，这使得我们可以更加简单的去观察和细调模型。同样的，这个时候损失函数的计算方法也需要我们去重新定义。这个时候我们只关注于正确类的那个概率数值，并把它取对数之后加上负号就可以得出结果了。如果说很有把握，那么这个数值就应该接近于0，如果预测正确类的概率非常低，那么这个数值就会非常非常大



### 梯度

我们构建的支持向量机（SVM）被表述为这样子的一个数学式子
$$
L = \frac{1}{N} \sum_i \sum_{j\neq y_i} \left[ \max(0, f(x_i; W)_j - f(x_i; W)_{y_i} + 1) \right] + \alpha R(W)
$$
我们发现，如果说样本的xi做出与真实标签yi一样的参数设置（这个就是开到大红出完美模型了），那么这个损失函数Loss的值也会变得非常非常小，那么，到底怎么去找出那个完美的W就是我们这里需要讨论的事情了。



通过数学分析的手段我们可以得出单个样本损失的表达式是这样的
$$
L_i = \sum_{j\neq y_i} \left[ \max(0, w_j^Tx_i - w_{y_i}^Tx_i + 1) \right]
$$
从公式里面我们就可以看出来了，每个样本的损失就是线性函数之和。并且W的每一行前的符号时而为正时而为负，正就是正确类别，负就是错误类别



这个时候要讲一个很经典的东西了就是那个**碗图**，这个东西可以帮助我们理解梯度到底是在干什么东西。

我们以W矩阵里面的单个权重为例子，考虑三个一维数据点和三个类别的简单数据集，那么完整的SVM损失就可以表示为（无正则化）：
$$
\begin{align} L_0 = & \max(0, w_1^Tx_0 - w_0^Tx_0 + 1) + \max(0, w_2^Tx_0 - w_0^Tx_0 + 1) \\\\ L_1 = & \max(0, w_0^Tx_1 - w_1^Tx_1 + 1) + \max(0, w_2^Tx_1 - w_1^Tx_1 + 1) \\\\ L_2 = & \max(0, w_0^Tx_2 - w_2^Tx_2 + 1) + \max(0, w_1^Tx_2 - w_2^Tx_2 + 1) \\\\ L = & (L_0 + L_1 + L_2)/3 \end{align}
$$
然后就可以画出来一个和碗一样的图片

这个图的横坐标向右就是不断地去提升权重的数值，纵坐标就是表示这损失函数值的大小。这么说可能很晦涩，晦涩就对了。

事实上这就是三个数据在不同的权重下的函数表达式，只是我们利用方法把他们合并在了一起，于是就得到了这么个碗的结构。由于我们需要得到的是一个面对很多数据都能够实现低损失的模型（也就是W），那么我们这个时候就需要考虑W的数值大小在哪里最合适。那么通过图就可以很直观的看出来在碗底的时候最合适。

这个时候我们要怎么调整参数，就可以引入到我们所说的梯度。如果这时候我们发现我们的模型数值大小在碗的左边，用一个很好懂的数学概念，大伙应该都学过，就是我们经常说的**斜率**

在左边的时候斜率比0小，那么这个时候就代表着我们需要把模型权重的数值加一加，那么就是减去这个比0小的斜率。如果在右边呢，斜率大于0，这说明我们就要减去这个比0大的斜率，而这个时候就引入到了我们今天要讲的核心概念——**梯度下降**

#### 梯度计算法

计算梯度有两种方法，一种是很慢但是很简单的方法，这种方法称之为数值梯度计算。另一种情况是很快，很准，但是很容易出错，这种方法的数学原理是基于微积分的，我们称之为解析梯度。

**使用有限差分法数值计算梯度**：

```python
def eval_numerical_gradient(f, x):
  """
  a naive implementation of numerical gradient of f at x
  - f should be a function that takes a single argument
  - x is the point (numpy array) to evaluate the gradient at
  """

  fx = f(x) # evaluate function value at original point
  grad = np.zeros(x.shape)
  h = 0.00001

  # iterate over all indexes in x
  it = np.nditer(x, flags=['multi_index'], op_flags=['readwrite'])
  while not it.finished:

    # evaluate function at x+h
    ix = it.multi_index
    old_value = x[ix]
    x[ix] = old_value + h # increment by h
    fxh = f(x) # evalute f(x + h)
    x[ix] = old_value # restore to previous value (very important!)

    # compute the partial derivative
    grad[ix] = (fxh - fx) / h # the slope
    it.iternext() # step to next dimension

  return grad
```

这种方法其实就是学高数的时候那种最简单最普通的方法，不准，但有效。导数的那个公式里面是要让h趋近于0，这种情况下其实就是去让h为一个非常小的数值，一般而言1e-5就已经非常小了

我们需要去考虑一个问题，我们对这个模型的数值进行微调的过程中我们要如何保证步长，不会迈太多也不会迈太少，步长是一个非常重要的超参数。

这个方法的效率非常非常低，假设我们的数据集是10*3073的一个矩阵，那么总共有多少个参数？答案是30730个参数，一次参数更新就需要改30730个参数，这是完全不可以接受的，所以我们就需要用到更好的方法



**通过微积分计算解析梯度**

数值梯度计算起来确实很简单，但是就是它又不准又繁琐。而利用微积分就可以实现相对简单的计算，代价是，容易出错。

以单个数据点的SVM损失函数为例子
$$
L_i = \sum_{j\neq y_i} \left[ \max(0, w_j^Tx_i - w_{y_i}^Tx_i + \Delta) \right]
$$
我们可以对权重进行微分，例如说对wyi求梯度，我们能得到这样的公式
$$
\nabla_{w_{y_i}} L_i = - \left( \sum_{j\neq y_i} \mathbb{1}(w_j^Tx_i - w_{y_i}^Tx_i + \Delta > 0) \right) x_i
$$
1是一个指示函数，当内部条件为真取1，否则就是0。这就很简单了

#### 梯度下降

既然我们已经知道了怎么计算梯度，那么梯度下降就是一个非常显而易见的事情了

```python
while True:
  weights_grad = evaluate_gradient(loss_fun, data, weights)
  weights += - step_size * weights_grad # perform parameter update
```

这个循环看似很简单，实际上是后面很多东西的基石

小批量梯度下降。在大规模应用（例如 ILSVRC 挑战赛）中，训练数据可能包含数百万个样本。因此，为了仅执行一次参数更新而计算整个训练集上的完整损失函数似乎是一种浪费。解决这一挑战的常用方法是基于训练数据的分批计算梯度。例如，在当前最先进的卷积神经网络中，一个典型批次包含来自 120 万总训练集中的 256 个样本。随后利用该批次执行参数更新：

```python
while True:
  data_batch = sample_training_data(data, 256) # sample 256 examples
  weights_grad = evaluate_gradient(loss_fun, data_batch, weights)
  weights += - step_size * weights_grad # perform parameter update
```



**总结**：梯度是什么，你可以理解为就是一个super big 的矩阵。我们首先要对原本的矩阵W求导，因为原本的矩阵会携带非常多参数，所以我们也会得到一个有非常多参数的求导结果，矩阵相减是要求矩阵的结构大小一致的，所以求导之后得到的结果其实就是一个完全一样结构的新矩阵。事实上这个新矩阵带有的参数是有正有负的，我们不用管，直接减就可以了。

这个时候其实就有一个问题，为什么我们要减去导数？我个人的理解是其实我们减去的并不是导数，而是1*导数。为什么要这么做。我们想一下，如果说我们对于这个矩阵减去一个固定的值，那么产生的影响大小是非常不一样的，导数是100和0.01的W看似都只是减了1，但是两个的影响会完全不一样。这样子就会导致我们的模型在判断不同类别的时候会时而准时而不准（其实感觉应该大概率是都不准）那么现在去思考一个问题，我们为什么要去把那个碗图里面的所有线都合成起来？

很简单，这个模型需要保证一个固定的loss损失，而我们进行调整，斜率大和斜率小我们调整的是完全不一样的因此我们就需要进行“按劳分配”。斜率大就说明这个地方对于loss的贡献大，那我们就要直接大刀阔斧的改，斜率小说明这个地方其实已经很准确了，不要再乱改了。

其实我说的那个1*导数前面的1倒也不是乱写的，那个东西叫做学习率，但这是后话了。

其实我觉得这个大梯度动大刀小梯度动小刀的东西非常反直觉，但其实我们是要利用大梯度去对模型大动刀，因为梯度大，说明模型已经错的很离谱了

题外话是，某种程度上我们的直觉是对的，因为动大刀确实有飞出去的风险，这就是后面的优化器的事情了
